org.nosco
Class Context

java.lang.Object
  org.nosco.Context

public class Context
extends java.lang.Object

This class is used for tweaking query executions.   Multiple context levels are supported (currently: vm, thread-group and thread) in increasing order of precedence.   A context can start and stop transactions, set default DataSources (optionally per package or class), and change what schema is referenced. Each context-altering method returns an Undoer object, which lets you undo the context change at a later date. (by calling Undoer.undo())   By default all Undoers undo themselves when they're GCed.   But you can suppress this behavior by calling Undoer.setAutoUndo(false). Typical use would be like the following (where db.DEFAULT is the data source generated by the CodeGenerator or pulled from a Query instance): 

   Context.getThreadContext().startTransaction(db.DEFAULT);
   SomeClass x = SomeClass.ALL.get(SomeClass.ID.eq(123));
   x.setName("my new name");
   x.update();
   doSomeOtherWork();
   SomeClass.ALL.where(SomeClass.ID.eq(456)).deleteAll();
   Context.getThreadContext().commitTransaction(db.DEFAULT);
Note that database calls (through this API) in doSomeOtherWork() will also be within the transaction (as long as they're still within the same thread).
